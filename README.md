# Daily Review · 每日复盘系统

> 通过循序渐进的提问帮助你回忆、澄清、表达，将零散信息整合为可积累的 Markdown 文档。

---

## 项目简介

Daily Review 是一个每日复盘与个人输出系统，帮助你：

- 通过循序渐进的提问回忆、澄清、表达
- 严格区分：事实 / 感受 / 认知 / 决策
- 将零散信息整合为可积累、可对比、可复用的 Markdown 文档
- 自动校验 TELOS 一致性与执行摩擦
- 附加自我觉察、思维边界、周度分析等系统级模块
- 所有内容以「可执行、低成本、真实」为第一原则

### 模块说明

| 模块 | 定位 | 时长 | 使用建议 |
|------|------|------|----------|
| **Awareness** | 系统运行状态检测（Debug Mode） | 1 分钟 | 可跳过，每个问题带引导示例 |
| **L1 快速版** | 核心 3 问，每日保底 | 3 分钟 | 推荐每天都做 |
| **L2 标准版** | 完整 7 问，推荐常态 | 10 分钟 | 状态好时做 |
| **Thinking** | 思维边界与原则过滤 | 1 分钟 | 可跳过，每日记录边界 |
| **L3 深度版** | 按需展开特定议题 | 按需 | 有感触时展开 |
| **Weekly Analysis** | 周度综合分析（仪表盘） | 5 分钟 | 仅周日使用 |

---

## 核心原则

1. **先对话，后输出** - 不替用户编造内容，通过提问引导回忆
2. **分层设计** - L1 快速版保底，L2 标准版完整，L3/Thinking/Weekly 按需展开
3. **严格区分四层** - 事实 / 感受 / 认知 / 决策
4. **可执行第一** - 所有调整必须低成本、真实
5. **自动校验** - 关注 TELOS 一致性与执行摩擦
6. **可持续** - 宁可每天做 L1，也不要断断续续做 L3
7. **可跳过** - 所有新增模块都支持跳过，不强制

## 使用

### 基础用法

在 Claude Code 中，通过以下方式使用：

```
/daily-review
```

系统会引导你完成复盘对话。

### 日期参数

```
/daily-review               # 复盘今天
/daily-review yesterday    # 复盘昨天
/daily-review 2025-01-30   # 复盘指定日期
/daily-review sunday       # 本周日，做 Weekly Analysis
```

### 对话方式选择

系统支持两种对话方式，开场时会让你选择：

- **逐个问**：一个问题答完再问下一个，适合放松时慢慢聊
- **一起呈现**：几个问题一起问，适合想快速答完的情况

可以说「逐个」「一起」切换模式。

### 跳过说明

所有可选模块都支持跳过，可以直接说：

- 「可以了」「够了」「跳过就跳过吧」「skip」「NA」「没」「没有」

## 模块详解

### Awareness · 自我觉察（Debug Mode）

**定位**：系统运行状态检测，不是复盘事实，不是分析结果。

**触发时机**：开场之后、正式复盘之前（可跳过）

**5 个核心问题**（每个问题附带引导示例）：

1. **Ego 驱动**：今天有哪些念头是由自尊/证明自己驱动的？
   - 引导示例：想在朋友圈展示什么、担心被人看不起、想证明自己的能力

2. **「应该」vs 想做**：今天做了哪些出于「应该」而不是真正想做的事？
   - 引导示例：应该陪客户吃饭、应该表现得合群、应该参加某个会议

3. **问题态度**：面对最重要的问题，是在改变/接受/逃避？
   - 改变＝在行动；接受＝放下/调整预期；逃避＝回避/拖延

4. **难以抉择**：是否有一个迟迟无法决定的选择？
   - 原则：难以抉择即否定，默认答案是什么？

5. **留白时间**：今天是否有 ≥10 分钟的留白？
   - 引导示例：散步、发呆、洗澡、一个人坐着

---

### Thinking · 思维边界

**定位**：建立保护边界，过滤杂讯，保护高价值输出。

**触发时机**：L2 复盘之后、L3 之前（可跳过）

**核心原则**：

- **价值密度判断**：做事前判断是否沉淀长期资产、推进核心主线，低价值事务应减少或拒绝
- **消除「应该」**：只做真正想做的事，不因社会期望/负罪感/规范产生行为
- **难以抉择即否定**：重大决策不确定时，答案就是「不」
- **留白时间**：拒绝无意义社交与会议，独处产生创造

**今日记录**：

- 今天是否有思维越界？（被无关信息牵走、被低价值事务打断）
- 今天是否有主动应用原则做出否定/拒绝/停止的决策？

---

### Weekly Analysis · 周度综合分析

**定位**：低频聚合模块，偏仪表盘风格，不追求精确只追求趋势感。

**触发时机**：
- 每周日使用（本周最后一次 Daily Review）
- 用户主动要求时（`/daily-review sunday` 或 `/daily-review weekly`）

**工作流程**：

1. **自动读取**：读取本周（周一至周日）所有 Daily 文档
2. **生成概览**：总完成天数、平均能量、高频情绪、主要产出
3. **四维度评分**：日常习惯执行度、知识输入质量、价值输出情况、思维边界守护程度
4. **归因选择**：输入不足 / 输出不足 / 边界失守 / 执行断裂
5. **用户补充**：本周亮点、本周最大教训

## 使用场景示例

### 场景 1：快速日常复盘（3 分钟）

**输入：**
```
/daily-review
```

**系统会问（逐个问模式）：**

1. 今天最关键的 1 件事是什么？整体感觉如何？
2. 今天情绪最高点或最低点是什么？（何时/发生了什么/几分 1-10）
3. 明天最重要的 1 件小事是什么？（如果明天只能花 15 分钟、不花钱，做什么最值？）

### 场景 2：完整复盘（10 分钟）

完成 L1 后，说「继续」「展开」进入 L2：

**L2 七组问题**：
- 第 1 组：三时段状态（早晨/下午/晚上）
- 第 2 组：今日日志（事实 × 感受分离）
- 第 3 组：目标推进（TELOS 映射 + 第一性追问）
- 第 4 组：输入输出（摘抄 + 理解 + 第一性拆解）
- 第 5 组：昨日对照（Review vs Execution）
- 第 6 组：明日调整（低成本启动 MVP）
- 第 7 组：TELOS 校验

### 场景 3：周日周度总结

**输入：**
```
/daily-review sunday
```

**系统会自动**：
1. 读取本周所有 Daily 文档
2. 生成四维度评分
3. 引导你选择本周主要归因
4. 让你补充亮点和教训

## TELOS 集成

系统会自动读取你的 TELOS 文件进行校验：

**读取逻辑**：
1. 扫描 `telos-dir` 配置的文件夹（默认：`...\\TELOS_Reviews`）
2. 解析文件名中的版本号（如 `TELOS_V1.0`、`TELOS_1.1`）
3. 选择版本号最新的文件
4. 在 TELOS 校验阶段引用该文件的 Goals、Strategies

**版本号判断**：
- `TELOS_V1.0` → 版本 1.0（较旧）
- `TELOS_1.1` → 版本 1.1（较新）
- 数值越大版本越新

**开场提示**：
> 已读取最新 TELOS 文件：`TELOS_X.X`（共 Y 个 Goals、Z 个 Strategies）

## 可选模块：AI 点评

在复盘完成后，可选择是否添加 AI 点评。

### 两份点评风格

**严格版（精英导师）**：
- 克制、有教养，但标准从不放松
- 用提问引导思考，不直接否定
- 点评维度：执行力、一致性、真实进度、问题解决、时间质量

**温暖版（理解者）**：
- 温暖、平和、肯定
- 强调坚持本身就很棒
- 点评维度：坚持、自觉、积累、真诚、可能性

## 核心原则速览

1. **L1 优先** - 保证每日可坚持，宁可每天 L1，不要断断续续做 L3
2. **不强制 L3** - 按需展开，用户没精力就停在 L2
3. **可跳过** - Awareness/Thinking/Weekly 都是可选的，不强制
4. **跳过短语** - 「可以了」「够了」「跳过就跳过吧」「skip」「NA」「没」「没有」
5. **不编造内容** - 信息缺失处标注「未发生 / 未提供 / 跳过」
6. **能量量化** - 🔋🔋🔋=高、🔋🔋=中、🔋=低
7. **≤3 原则** - 明日调整不超过 3 条
8. **实时反馈** - 告诉用户进度，让用户掌控节奏
9. **尊重选择** - 用户说「只做 L1」就只做 L1
10. **日期格式** - Daily 使用 `YYYY-MM-DD (周X)`，Weekly 使用 `YYYY-WXX`
11. **周日触发** - Weekly Analysis 仅在周日或用户主动要求时触发
12. **周度读取** - Weekly Analysis 会自动读取本周 Daily 文档进行总结
