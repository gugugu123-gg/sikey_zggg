# TELOS · Personal System Builder

> TELOS 希腊语，意思是「目的 / 终点 / 内在指向」。这是一套结构化自我访谈 + 个人系统建模框架。

---

## 项目简介

TELOS 不是目标管理工具，而是一套 **个人系统建模框架**，帮助你：

- 把「我是谁」「我想要什么」「什么在阻止我」转化为清晰的结构
- 建立可以被 AI 深度协作、持续优化的长期上下文
- 通过版本号管理，跟踪个人系统的演进阶段

---

## 核心特性

### 六层结构

```
┌─────────────────────────────────────────────────────┐
│                   TELOS 六层结构                     │
├─────────────────────────────────────────────────────┤
│  Problems（问题）    │  什么让我不爽                  │
│  Goals（目标）       │  我想改变什么                   │
│  Challenges（挑战）  │  什么在阻止我                   │
│  Strategies（策略）  │  我打算用什么方法               │
│  Projects（项目）    │  我现在在做什么                 │
│  Workflows（工作流） │  我每天实际上怎么运转           │
└─────────────────────────────────────────────────────┘
```

### 版本与阶段机制

- **V1.0 → V1.x**：阶段内迭代（小更新）
- **V1.x → V2.0**：进入新阶段（大更新）

**何时更新版本**：
- 新增重要 Projects
- Goals 达成或调整
- Strategies 重大变化
- 你觉得可以进入下一个阶段了

**版本号由你决定**，AI 在内容整理后询问你。

## 安装

### 方法一：通过 Git 克隆

```bash
# 克隆到 Claude Code 的 skills 目录
git clone https://github.com/[你的用户名]/telos.git ~/.claude/skills/telos
```

### 方法二：手动安装

1. 下载本项目的 ZIP 文件或克隆到本地
2. 将 `telos` 文件夹复制到 Claude Code 的 skills 目录：
   - **macOS/Linux**: `~/.claude/skills/`
   - **Windows**: `%USERPROFILE%\.claude\skills\`

3. 确保文件夹结构如下：
   ```
   ~/.claude/skills/telos/
   ├── SKILL.md       # 技能定义文件
   └── README.md      # 说明文档（本文件）
   ```

### 验证安装

重启 Claude Code 或重新加载 skills 后，在对话中输入：

```
/telos 建立
```

如果安装成功，该技能将被激活。

## 使用

### 基础命令

| 命令 | 用途 |
|------|------|
| `/telos 建立` | 首次建立 TELOS 系统 |
| `/telos 更新` | 阶段内迭代（小更新） |
| `/telos 升级` | 阶段升级（大更新） |
| `/telos 读取` | 查看当前 TELOS 内容 |

### 使用场景

#### 场景 1：首次建立 TELOS（3 分钟）

```
/telos 建立
```

**流程**：
1. 自由倾倒（什么问题让你不爽、想改变什么、什么在阻止你）
2. 结构化填充（六层结构）
3. 整理成文档，设定版本号

**输出**：`TELOS_V1.0.md`

#### 场景 2：阶段内迭代（2 分钟）

```
/telos 更新
```

**适用**：小调整，如新增项目、细化目标、优化工作流

**流程**：
1. 读取当前版本
2. 识别变化
3. 更新内容
4. 设定版本号（如 V1.0 → V1.1）

#### 场景 3：阶段升级（5 分钟）

```
/telos 升级
```

**适用**：进入新阶段，如某 Goals 达成、大量 Projects 更新

**流程**：
1. 阶段复盘（Goals 完成度、Problems 解决情况）
2. 判定是否进入新阶段
3. 迁移与更新（已完成的内容如何处理）
4. 设定新版本号（如 V1.x → V2.0）

#### 场景 4：查看当前 TELOS

```
/telos 读取
```

## 输出格式

### 文件命名

```
TELOS_Vx.x.md
```

例如：`TELOS_V1.0.md`、`TELOS_V1.1.md`、`TELOS_V2.0.md`

### 文件内容

```markdown
---
tags: [telos, personal-system]
date: YYYY-MM-DD
version: Vx.x
---

# TELOS · Personal System

## 版本信息

- 版本号：Vx.x
- 创建/更新时间：YYYY-MM-DD

## 一、Problems（问题）

- P1：xxx（状态：活跃/已解决）

## 二、Goals（目标）

- G1（对应 P1）：xxx（状态：进行中/已完成）

## 三、Challenges（挑战）

- 技术/资源/心理/系统障碍

## 四、Strategies（策略）

- 长期方法论

## 五、Projects（项目）

- 项目名：对应目标、当前阶段、核心价值

## 六、Workflows（工作流）

- 想法捕捉/学习流程/写作输出/编程开发/复盘迭代
```

- **TELOS 输出目录**：`D:\obsidian仓库\咕咕咕的知识管理\000myself\1system\TELOS_Reviews`
- **复盘读取目录**：`D:\obsidian仓库\咕咕咕的知识管理\000myself\1system\Journals`
- **文件名**：`TELOS_Vx.x.md`
- **版本号**：由用户自行输入
- **建议**：保留历史版本，方便追踪演进

**自定义路径**：编辑 `SKILL.md`，修改以下配置项：
- `output-dir`：TELOS 文件输出路径
- `journals-dir`：复盘文件读取路径

## 与 Daily Review 的集成

此 skill 与 `daily-review` skill 配合使用：

| Skill | 用途 |
|-------|------|
| `telos` | 建立/更新 TELOS 系统 |
| `daily-review` | 每日复盘，自动读取最新 TELOS 校验对齐 |

`daily-review` 会自动读取 `output-dir` 下最新版本的 TELOS 文件，在每日复盘的 TELOS 校验阶段引用其中的 Goals 和 Strategies。

## 可选模块：AI 点评（基于历史复盘记录）

在 TELOS 内容整理完成后，可选择是否添加 AI 点评。

### 功能说明

**触发时机**：TELOS 内容收集完成、设定版本号之后

**数据来源**：读取 `Journals` 目录下的 Daily/Weekly/Monthly 复盘文件

### 时间跨度选择

| 选项 | 说明 |
|------|------|
| 全部读取 | 从第一篇复盘到今天 |
| 从上次 TELOS 更新开始 | 从上次更新版本的时间点读到现在 |
| 自定义起始 | 指定开始日期 |
| 自定义跨度 | 如「最近 7 天」「最近 1 个月」 |

### 两份点评风格

#### 严格版

**角色**：精英导师

**特点**：
- 克制、有教养，但标准从不放松
- 用提问引导思考，不直接否定
- 像有阅历的前辈在复盘，不是审判
- 实事求是，不为了锐评而锐评

**点评维度**：执行力、一致性、真实进度、问题解决、时间质量

#### 温暖版

**角色**：理解者，提供情绪价值

**特点**：
- 温暖、平和、肯定
- 承认问题，但不过度渲染
- 强调坚持本身就很棒
- 让用户知道被看见

**点评维度**：坚持、自觉、积累、真诚、可能性

### 输出位置

两份点评会附加在 TELOS 文档末尾的「附录：AI 点评」章节。

| 场景 | 关键问法 |
|------|----------|

## 核心原则

1. **真实 > 好看** - 不要美化，如实记录
2. **具体 > 抽象** - 问题要具体，避免模糊
3. **版本由你定** - AI 只给建议，版本号由你输入
4. **定期复盘** - 建议至少每个月复盘一次阶段进展
5. **保留历史** - 旧版本不要删除，方便追踪演进
6. **AI 是工具** - TELOS 是给你自己用的，不是给 AI 看的
